## 遇到的问题和解决

1. 问题：订单状态有 5 种，UI 上如何清晰区分
   解决：结合后端状态码，用黄（待接单）/ 蓝（已接单 / 进行中）/ 绿（已完成）/ 灰（已取消）四色语义映射，与前端同学沟通对齐

2. 问题：UI 展示字段与后端 API 返回字段不一致
   解决：对照 docs/api.md，梳理各页面所需字段，与前端协商统一命名规范

## 心得体会

作为后端同学参与 UI 作业，最大收获是从用户视角理解了订单状态机的设计意图。
后端的状态流转（PENDING → ACCEPTED → IN_PROGRESS → CONFIRMING → DONE）
直接决定了前端每个状态下可操作的按钮，两端协同设计更利于减少后续联调成本。
