## 遇到的问题和解决

### 1. React Router 路由配置问题
**问题：** 初次配置路由时，底部导航栏在登录页也会显示，影响用户体验。

**解决：**
- 使用 useLocation hook 获取当前路径
- 创建 hideNavPaths 数组存储不需要导航栏的路径
- 根据当前路径动态控制导航栏的显示/隐藏
- 将路由逻辑封装在 AppContent 组件中，确保 useLocation 在 Router 内部使用

### 2. CSS 样式冲突问题
**问题：** 不同页面的 CSS 类名相同导致样式相互影响，出现样式污染。

**解决：**
- 为每个页面组件使用独立的 CSS 文件
- 使用特定的命名前缀（如：login-container, task-list-container）
- 采用 BEM 命名规范，提高样式的可维护性
- 确保全局样式只包含通用重置样式

### 3. 表单状态管理问题
**问题：** 任务发布页面表单字段较多，useState 管理多个状态显得冗余。

**解决：**
- 使用单个 state 对象管理整个表单数据
- 创建通用的 handleInputChange 函数处理所有输入
- 通过 name 属性动态更新对应字段
- 使用扩展运算符保持状态的不可变性

### 4. 移动端适配问题
**问题：** 部分组件在小屏幕设备上显示不完整或布局错乱。

**解决：**
- 使用 CSS Media Queries 实现响应式布局
- 设置断点 @media (max-width: 768px)
- 调整移动端的字体大小、间距和布局方向
- 使用 flexbox 和 grid 实现灵活布局

### 5. 页面过渡动画卡顿
**问题：** 页面切换时动画不流畅，用户体验较差。

**解决：**
- 使用 CSS transition 替代 JavaScript 动画
- 优化 transition 属性，只对必要的属性添加过渡
- 使用 transform 替代 top/left 等属性实现位移动画
- 添加 will-change 属性提示浏览器进行优化

## 心得体会

在本次 HelpMate 校园跑腿平台的架构设计和开发过程中，我深刻体会到了以下几点：

### 1. 架构设计的重要性
在项目初期花时间进行充分的架构设计是非常值得的。清晰的前后端分离架构、合理的组件划分、规范的目录结构，都为后续的开发工作奠定了良好的基础。特别是在页面开发阶段，预先设计好的组件复用策略大大提高了开发效率。

### 2. 技术选型需要权衡
在选择技术栈时，不能盲目追求新技术，而是要根据项目实际需求和团队能力进行权衡。例如，我们选择 React 而不是 Vue，是因为 React 的生态更丰富，组件化思想更彻底，更适合这种中等复杂度的项目。

### 3. 用户体验至关重要
在实现 UI 设计稿时，我深刻理解到细节决定成败。按钮的悬停效果、表单的验证反馈、页面的加载状态，这些看似微小的交互细节，实际上对用户体验有着重要影响。

### 4. 文档的价值
编写 architecture.md、database.md 等文档的过程，不仅帮助团队成员快速了解项目，也促使我对架构设计进行更深入的思考。好的文档是项目可维护性的重要保障。

### 5. 问题解决能力的提升
在开发过程中遇到的各种问题，从路由配置到样式冲突，从状态管理到性能优化，每一个问题的解决都是一次学习和成长的机会。通过查阅文档、分析源码、实验验证，逐步建立起解决问题的思维模式。

### 6. 代码质量意识
通过遵循命名规范、保持代码简洁、注重代码复用，我培养了良好的代码质量意识。虽然项目规模不大，但养成良好的编码习惯对今后的职业发展非常重要。

### 7. 团队协作的重要性
软件开发是团队协作的过程，通过 Git 进行版本控制、通过 PR 进行代码审查、通过文档进行知识共享，这些实践让我认识到团队协作的重要性。

总的来说，这次项目让我从理论走向实践，从设计走向实现，是一次非常宝贵的学习经历。未来我会继续深入学习前端技术，提升自己的工程能力。
