# 设计说明

> 项目：HelpMate — 校园跑腿 / 互助平台
> 平台规范：Material Design 3（Android-first，兼容微信小程序）
> 技术栈：React Native / 小程序 + Spring Boot

---

## 一、用户研究

### 目标用户群体

HelpMate 面向**校园内有跑腿或互助需求的学生群体**，主要分为两类：

| 用户类型 | 特征 |
|----------|------|
| 发单者（需求方） | 需要帮忙取快递、送餐、代购、翻译文件等校园日常事务 |
| 接单者（服务方） | 有空余时间，愿意通过帮助他人赚取报酬或积分 |

典型用户：在校大学生，年龄 18–25 岁，熟悉手机 App 和微信小程序操作，有移动支付习惯。

### 核心用户场景

**场景 1：发布跑腿任务**
用户小明在宿舍不方便外出，打开 HelpMate → 点击"发布任务" → 填写任务内容（取快递、地址、时间要求）、设置报酬 → 发布成功后等待接单者响应。

**场景 2：抢单并完成任务**
用户小红打开首页地图，看到附近任务标注 → 筛选"取快递"分类 → 点击任务查看详情 → 点击"立即接单"→ 实时导航至取件地点 → 完成后确认交付，系统结算报酬。

**场景 3：任务状态跟踪**
发单者小明进入"我的订单" → 查看任务状态（待接单 → 已接单 → 配送中 → 已完成）→ 接单者到达后查看实时位置 → 收货后点击"确认完成"并评价。

**场景 4：钱包充值与提现**
接单者小红进入"钱包" → 查看余额和收益明细 → 申请提现到微信 / 支付宝。

### 竞品分析（可选）

| 竞品 | 优势 | 不足 |
|------|------|------|
| 饿了么跑腿 | 品牌知名度高，覆盖广 | 不专注校园，无互助属性 |
| 闲鱼帮帮 | 社区氛围好 | 无实时位置，状态跟踪弱 |
| 校内小程序 | 贴近校园 | 功能单一，无支付闭环 |

**HelpMate 差异化**：实时位置 + 订单状态机 + 校园钱包支付三者结合。

## 二、信息架构

```
HelpMate
├── 登录 / 注册
│   ├── 登录页
│   └── 注册页
├── 首页（地图 + 任务列表）
│   ├── 地图视图（实时任务标注）

├── 发布任务

│   ├── 选择时间要求
│   └── 确认发布 + 支付押金
├── 订单中心
│   ├── 我发布的（发单记录）
│   │   └── 任务状态追踪（含接单者实时位置）

    ├── 钱包（余额 / 充值 / 提现 / 明细）
    ├── 评价记录
    ├── 设置
    │   ├── 修改密码
    │   └── 通知设置
    └── 关于
```

## 三、配色方案

遵循 Material Design 3 动态配色体系，主色选用活力蓝，强调色选用校园绿，传达可靠、活力的品牌感。

| 角色 | 色值 | 用途 |
|------|------|------|
| 主色（Primary） | `#1A73E8` | 主按钮、地图标注、导航激活态 |
| 辅助色（Secondary） | `#5F6368` | 次要文字、图标、分割线 |
| 强调色（Accent）| `#34A853` | 成功状态、"已完成"标签、钱包收益 |
| 警示色（Warning） | `#FBBC04` | 待接单状态标签 |
| 背景色（Background） | `#F8F9FA` | 页面底色 |
| 表面色（Surface） | `#FFFFFF` | 卡片、对话框 |
| 错误色（Error） | `#EA4335` | 表单错误、超时预警 |
| 主色文字 | `#FFFFFF` | 主色背景上的文字 |

### 暗色模式（Dark Mode）

| 角色 | 色值 |
|------|------|
| 主色 | `#8AB4F8` |
| 强调色 | `#81C995` |
| 背景色 | `#121212` |
| 表面色 | `#1E1E1E` |
| 主要文字 | `#E8EAED` |

## 五、核心页面

### 1. 登录页（login）
用户输入用户名和密码登录，支持微信快捷登录，底部提供注册入口。
- 组件：Logo + 应用名"HelpMate"、用户名输入框、密码输入框（可显示/隐藏）、登录按钮、微信登录按钮、注册链接

### 2. 注册页（register）
新用户填写用户名、邮箱、密码、确认密码。
- 组件：4 个输入框、注册按钮、返回登录链接
- 校验：密码 ≥ 8 位，两次一致，邮箱格式合法

### 3. 首页（home）
地图视图 + 任务列表双模式，展示附近跑腿任务。
- 顶部：搜索框 + 地图/列表切换 Tab
- 地图模式：高德/百度地图，任务以彩色气泡标注
- 列表模式：分类 Chip（全部 / 取快递 / 送餐 / 代购 / 互助）+ 任务卡片列表
- 卡片内容：任务标题、报酬金额（醒目显示）、距离、截止时间、状态标签
- 底部导航：首页 / 发布 / 订单 / 个人中心

### 4. 任务详情页（detail）
展示单条任务完整信息，提供接单入口。
- 组件：任务标题、分类标签、报酬、截止时间、任务描述、取件/目标地址（地图缩略图）、发单者头像+昵称+评分
- 底部固定："立即接单"按钮（主色）

### 5. 发布任务页（create）
填写并发布新的跑腿/互助任务。
- 组件：任务标题（50 字）、分类选择、详细描述（500 字）、地址输入（地图选点）、报酬设置（滑块 + 手动输入）、截止时间选择器
- 顶部："发布"按钮，点击后进入支付确认页

### 6. 订单状态追踪页（order-track）
发单者实时查看任务进度和接单者位置。
- 顶部：订单状态步骤条（待接单 → 已接单 → 进行中 → 待确认 → 已完成）
- 中部：地图（接单者实时位置蓝点 + 目的地标注）
- 底部：接单者头像、昵称、距离预计到达时间；"确认完成"按钮（任务完成后激活）

### 7. 个人中心页（profile）
用户信息、钱包余额、订单记录入口。
- 顶部：头像、用户名、信用评分
- 功能入口：钱包、我的订单、评价记录、设置
- 钱包卡片：余额（大字显示）、充值 / 提现快捷按钮

## 六、交互说明

| 触发 | 行为 |
|------|------|
| 点击首页任务卡片 / 地图气泡 | 跳转任务详情页 |
| 点击详情页"立即接单" | 弹出确认对话框 → 确认后接单成功，跳转到订单导航页 |
| 点击底导"发布" | 跳转发布任务页 |
| 发布页点击"发布" | 跳转支付确认 → 支付成功后 Snackbar 提示，返回首页 |
| 订单页点击"确认完成" | 弹出评价弹窗，提交后结算报酬 |
| 首页下拉 | 刷新任务列表 |
| 列表滚动到底 | 自动分页加载（每页 10 条）|
| 地图拖动 | 加载当前视野内任务气泡 |
| 悬停按钮（Web/PC） | 背景色加深 8%，cursor: pointer |
| 表单输入错误 | 输入框下方红色提示文字 |
| 点击退出登录 | 确认对话框 → 清除 Token → 返回登录页 |

## 七、组件规范

- **按钮**：圆角 20dp，高度 48dp，文字 14sp Medium
- **任务卡片**：圆角 12dp，elevation 1，内边距 16dp，报酬文字用强调色加粗
- **状态标签 Chip**：圆角 16dp，高度 28dp，颜色对应订单状态（黄/蓝/绿/灰）
- **地图气泡**：圆形，直径 36dp，内含报酬金额，颜色同状态标签
- **输入框**：Material Outlined TextField，圆角 4dp
- **底部导航**：4 个 Tab，高度 56dp，图标 + 文字
- **Snackbar**：底部弹出，3 秒后消失
