校园跑腿小程序页面设计说明
一、页面概述
本次设计包含 订单进度页、个人中心页、任务详情页 三个核心页面，均服务于校园跑腿互助场景，以清晰的信息层级和友好的交互体验为设计目标。

二、页面设计详情

1. 订单进度页
页面定位：展示用户发布 / 参与的所有任务状态，方便快速追踪任务进展。
布局结构：
顶部导航栏：包含「订单进度」「消息列表」两个标签页，右侧显示未读消息角标，便于用户切换查看不同类型信息。
任务卡片列表：以卡片形式呈现每个任务，卡片内包含任务名称、状态标签、发布者、接单⼈、赏⾦、创建时间等核心信息，信息分组清晰，便于快速浏览。
状态标签：使用不同颜色区分任务状态（进行中 = 蓝色、待接单 = 橙色、已完成 = 绿色），视觉辨识度高，用户可一眼识别任务进度。
设计亮点：卡片式布局提升信息可读性，状态标签直观高效，符合移动端浏览习惯。
3. 个人中心页
页面定位：展示用户个人信息与发布 / 接取的任务列表，是用户管理个人任务的核心入口。
布局结构：
顶部用户信息区：包含头像、用户名、手机号（脱敏显示）、评分，右侧提供「设置」按钮，方便用户管理账号。
任务标签栏：分为「我发布的」「我接取的」两个标签，用户可快速切换查看不同角色的任务。
任务列表：展示任务名称、赏⾦、状态、接单⼈信息，提供「查看详情」「联系接单⼈」等操作按钮，满足任务管理需求。
设计亮点：信息层级分明，用户信息与任务管理分区清晰，操作按钮突出，便于用户快速执行操作。
4. 任务详情页
页面定位：展示单个任务的完整信息，供用户了解任务细节并决定是否接取。
布局结构：
顶部导航栏：包含返回按钮与页面标题，左侧返回按钮便于用户回溯上一级页面。
任务基础信息区：展示任务状态、发布时间、截止时间，以及任务名、任务描述、取件地点、送达地点等核心内容，信息完整且结构化。
发布者信息区：展示发布者头像、用户名、评分，提供「私信」按钮，便于用户沟通。
操作区：突出显示赏⾦金额，底部放置「接取任务」主按钮，引导用户完成核心操作。
设计亮点：信息全面且重点突出，发布者信用评分增强信任，操作按钮醒目，符合任务接取的业务流程。
三、设计原则
一致性：三个页面采用统一的蓝色主题色、卡片式布局、字体样式，保证整体视觉风格一致，提升用户体验连贯性。
可读性：信息分组清晰，关键信息（如赏⾦、状态）突出显示，便于用户快速获取核心内容。
易用性：操作按钮位置固定、样式醒目，导航逻辑清晰，降低用户学习成本，符合移动端交互习惯。
直观性：通过颜色区分任务状态，通过标签页分类信息，让用户无需思考即可理解页面内容。
